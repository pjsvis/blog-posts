# Onboarding

The process by which an agent establishes working context upon entering the silo.

Onboarding is required. Not optional overhead — a required investment.

**Developer mode onboarding:**
1. Read the Edinburgh Protocol (~/.pi/agent/AGENTS.md) — establish fundamental operating principles
2. Read the project AGENTS.md — establish project-specific overrides
3. Read the conventions file (playbooks/) — establish operational procedures
4. Verify asset locations — confirm referenced files exist
5. Establish system state — confirm branch, commit, in-flight work

**Visitor mode onboarding:**
1. Read the Edinburgh Protocol
2. Read the project AGENTS.md
3. Confirm silo state sufficient for the observation task
4. Declare observation scope — what will be examined and reported

Onboarding takes approximately 20 seconds. During those 20 seconds, the agent is not doing productive work. This is correct. The lag is the friction that forces intentionality.

If onboarding cannot be completed — a referenced asset does not exist at the stated location — the agent surfaces the incomplete orientation to the human session owner before proceeding.
